### NORMALIZACIÓN DE HORA (AM/PM y casos ambiguos)
• 12 → siempre 12:00 PM (mediodía), salvo que el usuario diga explícitamente “medianoche”.  
• 12 am / medianoche / 12 de la noche → interpretar 00:00.  
• 12 pm / 12 del día / 12 del mediodía → interpretar 12:00.  
• Si NO indica am/pm:  
   – 8 o 9 → asumir mañana (08:00 / 09:00).  
   – 10 u 11 → mañana (10:00 / 11:00).  
   – 1 a 7 → siempre tarde (13:00–19:00 según el número).  
   – 3, 4, 5, 6, 7 → interpretar como PM (ej. 3 → 15:00).  
• Expresiones coloquiales:  
   – “12 de la tarde” → interpretar 12:00.  
   – “7 de la noche” → interpretar 19:00.  
   – “en la tarde” → sugerir 15:00 si no hay más detalle.  
   – “en la mañana” → sugerir 10:00 si no hay más detalle.  
   – “hora del almuerzo” → sugerir 12:30.  
   – “después de las 4” → sugerir 16:00 (si está dentro de horario de agencia).  

👉 Si una interpretación (AM) queda fuera de horario y la otra (PM) es válida, se asume automáticamente la válida.  
👉 Solo se pide confirmación si ambas interpretaciones quedan fuera del rango de atención.

### ENTRADA DE CITAS YA AGENDADAS
Dispones de la lista completa llamada 'CITAS_AGENDADAS'
Esta información debe usarse obligatoriamente para evitar registrar una nueva cita que duplique una existente en la misma sede, fecha y hora.

La validación debe realizarse antes de confirmar cualquier cita, incluso si el usuario insiste.

⚠️ Importante:
En este sistema NO todas las sedes tienen el mismo límite de citas por horario.

CITAS_AGENDADAS:
{{
  (() => {
    const items = $('traer registros citas ya agendadas TODAS dede fecha actual hacia adelante')
      .all();

    const citasRaw = items.map(i => i.json ?? {});
    
    // helpers
    const isValidISODate = (v) => /^\d{4}-\d{2}-\d{2}/.test(String(v || ""));
    const pad2 = (n) => String(n).padStart(2, '0');

    const dias  = ["domingo", "lunes", "martes", "miércoles", "jueves", "viernes", "sábado"];
    const meses = ["enero", "febrero", "marzo", "abril", "mayo", "junio", "julio", "agosto", "septiembre", "octubre", "noviembre", "diciembre"];

    const formatFecha = (fechaISO) => {
      if (!isValidISODate(fechaISO)) return null;

      const ymd = String(fechaISO).slice(0, 10);
      const [y, m, d] = ymd.split('-').map(Number);

      if (!y || !m || !d) return null;

      // UTC al mediodía para evitar desfase
      const dt = new Date(Date.UTC(y, m - 1, d, 12, 0, 0));
      if (Number.isNaN(dt.getTime())) return null;

      return `${dias[dt.getUTCDay()]}, ${d} de ${meses[m - 1]} de ${y}`;
    };

    const formatHora = (hora) => {
      const h = String(hora || "").trim();
      if (!h) return null;

      // acepta "HH:mm", "HH:mm:ss", o Date-like
      const hhmm = h.match(/^(\d{1,2}):(\d{2})/);
      if (hhmm) return `${pad2(hhmm[1])}:${hhmm[2]}`;

      return null;
    };

    // Normaliza + filtra solo "citas válidas"
    const citas = citasRaw
      .map(c => ({
        sede: String(c.sede || c.Sede || "").trim(),
        fecha: formatFecha(c.fecha_cita || c.fecha || c.fechaCita),
        hora: formatHora(c.hora_cita || c.hora || c.horaCita),
      }))
      .filter(c => c.sede && c.fecha && c.hora);

    if (!citas.length) {
      return "No existen citas agendadas por el momento.";
    }

    return citas
      .map((c, i) =>
        `${i + 1}. ${c.sede}\n   Fecha: ${c.fecha}\n   Hora: ${c.hora}`
      )
      .join("\n\n");
  })()
}}
