SimpliaLeads — Historical Trends Tab
Developer Report
Purpose: define the architecture, KPI scope, visual logic, and implementation separation for the new “Historical Trends” tab connected to Supabase.
Module
Historical Trends tab
Data source
Supabase (daily snapshots, monthly visualization)
Primary time grain
Month (default view: last 12 months)
1. Module objective
The Historical Trends tab is not an operational real-time view. It is a longitudinal analysis layer designed to read structural performance across months and years.
It must be used to compare periods, identify seasonality, detect sustained deterioration or improvement, and export historical performance data using clean monthly views.
2. Required architectural separation
Operational tabs continue to read live or near-real-time data from Chatwoot for execution and daily management.
The Historical Trends tab must query Supabase only. It should not recompute historical performance directly from Chatwoot on each render.
This separation is required for performance, consistency, easier year-over-year comparison, and stable monthly KPI definitions.
3. Recommended historical data structure
Layer A — Daily snapshot table: one record per day with the base metrics required for historical analysis.
Layer B — Monthly aggregation layer: query, materialized view, or summarized table built from the daily snapshot table.
Layer C — Front-end consumption layer: pre-structured payloads for cards, monthly charts, segment views, and exports.

10. Visualization rules for medium-sized SMEs
Default filter window: last 12 months.
Extended comparison window: 24 months.
Maximum recommended visible range in one chart: 36 months.
Use line charts for rates and time-based performance metrics.
Use bar charts for month-end backlog and other discrete monthly volumes.
Avoid more than 2–3 series in one chart.
Use reference lines for target-based KPIs such as SLA compliance or desired conversion.
Avoid pie charts and over-colored visuals for historical analysis.

14. Final product decision
The Historical Trends tab should be implemented as a monthly historical analysis module, powered by Supabase, separate from the real-time Chatwoot tabs, and designed for long-range comparison, seasonality reading, and structural performance analysis.
